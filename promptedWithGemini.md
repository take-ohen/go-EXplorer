解説
.gitignore は、Gitのバージョン管理システムに対して、追跡（track）を意図的に無視させたいファイルやディレクトリを指定するためのファイルです。

/go-EXplorer: go build によって生成される実行ファイルです。これは開発者の各環境でソースコードから生成されるべきものであり、リポジトリに含めません。
*.test, *.out: go test コマンドで生成されるテスト用のバイナリやカバレッジプロファイルです。
/vendor/: Go Modules導入以前に使われていた依存ライブラリの管理ディレクトリです。現在は go.mod で管理するため、通常は無視します。
.vscode/, .idea/: VSCodeやGoLandなどのエディタが使用する設定ファイルです。個人の開発環境に依存するため、共有しません。
.DS_Store, Thumbs.db: macOSやWindowsが自動生成するシステムファイルです。プロジェクトとは無関係なため無視します。
.env: APIキーなどの機密情報や環境ごとの設定を記述するファイルです。セキュリティ上の理由からリポジトリに含めてはいけません。

package core

import (
	"os"
	"path/filepath"
	"sync"
)

// FileItem represents a file or directory found during scanning.
type FileItem struct {
	Path  string
	Name  string
	IsDir bool
	Size  int64
}

// Scanner provides high-speed directory scanning capabilities.
type Scanner struct {
	// TODO: Implement channels for async results
	// sem limits the number of concurrent goroutines to prevent resource exhaustion.
	sem chan struct{}
}

// NewScanner creates a new Scanner instance with a concurrency limit.
func NewScanner() *Scanner {
	return &Scanner{}
	// Limit concurrency to 32 to avoid "too many open files" errors.
	return &Scanner{
		sem: make(chan struct{}, 32),
	}
}

// Scan starts scanning the directory at root.
// It returns a channel that receives FileItem objects as they are found.
// The channel is closed when the scan is complete.
func (s *Scanner) Scan(root string) <-chan FileItem {
	out := make(chan FileItem, 1000) // Buffered channel for performance
	var wg sync.WaitGroup

	wg.Add(1)
	go func() {
		defer wg.Done()
		s.walk(root, out, &wg)
	}()

	// Monitor routine to close the channel when all workers are done.
	go func() {
		wg.Wait()
		close(out)
	}()

	return out
}

func (s *Scanner) walk(path string, out chan<- FileItem, wg *sync.WaitGroup) {
	// Acquire semaphore token
	s.sem <- struct{}{}
	defer func() {
		// Release semaphore token
		<-s.sem
	}()

	entries, err := os.ReadDir(path)
	if err != nil {
		// Permission denied or other errors are ignored in this prototype.
		return
	}

	for _, entry := range entries {
		fullPath := filepath.Join(path, entry.Name())

		// Get file info for size
		info, err := entry.Info()
		var size int64
		if err == nil {
			size = info.Size()
		}

		out <- FileItem{
			Path:  fullPath,
			Name:  entry.Name(),
			IsDir: entry.IsDir(),
			Size:  size,
		}

		if entry.IsDir() {
			wg.Add(1)
			go func(p string) {
				defer wg.Done()
				s.walk(p, out, wg)
			}(fullPath)
		}
	}
}

scanner.go の役割
このファイルは、指定されたディレクトリを高速にスキャンするためのコアロジックです。

目的: Goの並行処理機能（Goroutine）を使い、大量のファイルやディレクトリを効率的に探索する。
主要な構成要素:
FileItem 構造体: 見つかったファイルやディレクトリの情報を保持する。
Scanner 構造体: スキャン処理全体を管理する。
Scan() メソッド: スキャンを開始し、結果をリアルタイムで送り返すための「通り道」を返す。
walk() メソッド: 実際にディレクトリを再帰的に探索する内部関数。
<- (チャネル演算子) の解説
<- は、Goの並行処理で使われるチャネル (Channel) を操作するための演算子です。 チャネルは、Goroutine間で安全にデータを送受信するための「パイプ」や「通り道」のようなものです。

この演算子は、記述する位置によって2つの意味を持ちます。

1. channel <- data (送信)
意味: チャネルにデータを送信する。

scanner.go の walk 関数内に、この使い方の例があります。

go
// c:\Users\dora\home\src\go\go-EXplorer\internal\core\scanner.go
// ...
		out <- FileItem{
			Path:  fullPath,
			Name:  entry.Name(),
			IsDir: entry.IsDir(),
			Size:  size,
		}
// ...
構文分解:
out: chan<- FileItem 型のチャネル（データの通り道）。
<-: 送信演算子。
FileItem{...}: 送信するデータ。
図解:
text
+----------------+      +----------------+
| FileItemデータ |----->|  out チャネル  |
+----------------+      +----------------+
      (送信)
解説: walk 関数は、見つけたファイルの情報 (FileItem) を out という名前のチャネルに送信しています。これにより、スキャン処理の完了を待たずに、見つけたものから順次結果を外部に伝えることができます。
2. <- channel (受信)
意味: チャネルからデータを受信する。

scanner.go の walk 関数内に、この使い方の例があります。

go
// c:\Users\dora\home\src\go\go-EXplorer\internal\core\scanner.go
// ...
	// Acquire semaphore token
	s.sem <- struct{}{}
	defer func() {
		// Release semaphore token
		<-s.sem
	}()
// ...

<-: 受信演算子。
s.sem: chan struct{} 型のチャネル。
図解:
text
+----------------+      +----------------+
|  (空のデータ)  |<-----|  s.sem チャネル|
+----------------+      +----------------+
      (受信)
解説: <-s.sem は、s.sem チャネルからデータを受信しようとします。このコードでは、同時に実行できるGoroutineの数を制限するために使われています。
s.sem <- struct{}{}: s.sem チャネルにデータを送信し、実行権を1つ消費する。
<-s.sem: s.sem チャネルからデータを受信し、実行権を1つ返却する。
3. 型定義における <- (チャネルの方向)
型定義で使う場合、<- はチャネルが「送信専用」か「受信専用」かを示します。

<-chan FileItem: 受信専用チャネル。このチャネルからはデータを受け取ることしかできない。
chan<- FileItem: 送信専用チャネル。このチャネルにはデータを送ることしかできない。

// Scanメソッドは「受信専用」チャネルを返す
func (s *Scanner) Scan(root string) <-chan FileItem {
// ...
}

// walkメソッドは「送信専用」チャネルを引数に取る
func (s *Scanner) walk(path string, out chan<- FileItem, wg *sync.WaitGroup) {
// ...
}
解説: Scan メソッドを呼び出す側は、結果を受信するだけです。誤ってデータを送信してしまうことを防ぐために、型レベルで「受信専用 (<-chan)」と明記しています。 逆に walk 関数は、結果を送信するだけなので「送信専用 (chan<-)」と明記されています。 このように方向を限定することで、より安全なプログラムを書くことができます。
